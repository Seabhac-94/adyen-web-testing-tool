Before you begin

The Adyen Tech Support Integrations tool is designed to allow you to test the full capacity of the Drop-in and Components, without having to make changes to code. It also allows you to test certain error scenarios.

Error recreation: Below is a description of errors you can recreate and the required parameters for them.

noInvoiceLines: a common error seen in PM's such as Klarna or Oney. Each PM has required data in a /payments request, and certain PM's require lineItems. If these aren't supplied, the error will be thrown. Test this on any version.

threeDS2Timeout: A main reason for this is due to the incorrect origin being sent in the /payments request. The payment will eventually go through, but once the SCA has been completed, it can take up to 10 minutes for onAdditionalDetails to be called by the component. To test this use a version v4.9.0 or lower and API v67. The issue is no longer present on v5.

noBrowserInfoProvided: browserInfo is a required parameter in the /payments request for 3DS2 transactions. If you open the console before starting the transaction, once you hit pay, you will see something like POST https://checkoutshopper-test.adyen.com/checkoutshopper/v1/submitThreeDS2Fingerprint?token=client_key 500 followed by Service at https://checkoutshopper-test.adyen.com/checkoutshopper/v1/submitThreeDS2Fingerprint?token=client_key is not available. This is the internal details call failing. Looking in the logs you will see an error similar to No Browser info provided. Test on any version using a 3DS2 test card.

invalidLanguageTag: the browserInfo requires certain parameters, and if any are missed or incorrectly sent, this can cause an issue. If you run this test and look in the "Request" container on the right, you will see that browserInfo.language: "incorrectL4ngu4ge" to simulate the incorrect format. Looking in the logs you will see a 422 error specifying the format it should follow.
